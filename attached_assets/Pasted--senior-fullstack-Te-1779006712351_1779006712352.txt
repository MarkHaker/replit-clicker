### 🔹 1. Логика бота (Python)
- При получении команды `/start` бот отправляет сообщение: `«Привет, это моя игра Кликер»`.
- Под сообщением размещается **ReplyKeyboard** (не Inline) со следующими кнопками:
  - `🎮 Играть` → открывает WebApp по ссылке
  - `📊 Мой прогресс` → выводит краткую сводку без захода в игру (монеты, сила клика, авто-доход/сек, уровень, реф. баланс)
  - `👥 Реферальная программа` → генерирует уникальную ссылку и показывает статистику приглашений
  - `💸 Вывести реф. бонусы` → конвертирует накопленный реферальный баланс в игровую валюту (соотношение 1:1) с анимацией/подтверждением
- **Обязательная проверка username:** Если у `message.from_user.username` значение `None`, бот блокирует все команды кроме `/start` и просит установить username в настройках Telegram. После установки проверка повторяется.
- **Реферальная система (глубокие ссылки + трекинг):**
  - Ссылка генерируется в формате: `https://t.me/ТвойБотUsername?start=ref_{user_id}`
  - При запуске бота по ссылке парсится `ref_{id}`, связь сохраняется в локальной базе.
  - **Механика начисления:** 35% от **всех кликов** приглашённого пользователя зачисляются на отдельный `ref_balance` пригласителя. Балансы строго разделены: `main_balance` (игра) и `ref_balance` (реферальный).
  - Данные хранятся в `data.json` или `sqlite` (на выбор, но с автобэкапом и безопасной записью).
  - Пример структуры данных:
    ```json
    {
      "user_12345": {
        "main_balance": 1500,
        "click_power": 2,
        "auto_income": 5,
        "ref_balance": 320,
        "invited_by": "user_98765",
        "invited_users": ["user_111", "user_222"],
        "last_save": 1718000000
      }
    }
    ```

### 🔹 2. Веб-приложение (Frontend + игровая логика)
- Стек: чистые `HTML`, `CSS`, `JavaScript` (без фреймворков и сборщиков, чтобы легко деплоить на GitHub Pages).
- Адаптивный mobile-first дизайн, оптимизированный под Telegram WebApp SDK.
- **Детальные механики кликера:**
  1. **Основной клик:** Большая интерактивная кнопка/изображение. При нажатии начисляются монеты, появляется анимация нажатия и всплывающие числа `+X`. Поддержка вибрации через `Telegram.WebApp.HapticFeedback.impactOccurred()`.
  2. **Магазин улучшений:** Вкладка с тремя категориями:
     - `Сила клика` (увеличивает монеты за одно нажатие)
     - `Авто-кликер` (пассивный доход в секунду)
     - `Множитель дохода` (умножает весь текущий доход)
     Каждое улучшение имеет уровни, экспоненциально растущую стоимость и визуальный индикатор прогресса. Кнопки покупки блокируются, если монет недостаточно.
  3. **Офлайн-прогресс:** При загрузке WebApp сравнивается текущее время с `lastSaveTime`. Рассчитывается: `время_отсутствия (сек) × авто_доход_в_сек`. Выдаётся накопленный бонус с анимацией и возможностью забрать его вручную.
  4. **Система сохранений:** Прогресс хранится в `localStorage` (JSON-структура: монеты, уровни улучшений, время последнего сохранения, пользовательский ID). Данные привязываются к `initDataUnsafe.user.id` из Telegram WebApp SDK. При закрытии вкладки или раз в 30 сек отправляй сводку кликов в бот через `Telegram.WebApp.sendData()` в формате `JSON.stringify({type: "sync", clicks: newClicks, timestamp: Date.now()})`. Бот обрабатывает данные, начисляет 35% реферального бонуса владельцу ссылки и обновляет `lastSaveTime`.
  5. **UI/UX:** 
     - 3 вкладки: `🖱 Кликер`, `🛒 Магазин`, `📊 Статистика`
     - Автоматическая подстройка темы под Telegram (`tg-theme-bg-color`, `tg-theme-button-color` и т.д.)
     - Счётчик монет с разделителями тысяч (например, `1.25M`)
     - Прогресс-бары до следующего уровня улучшений
     - Тултипы с описанием каждого апгрейда
     - Плавные CSS-анимации, отсутствие лагов на слабых устройствах

### 🔹 3. Архитектура и взаимодействие
- Поскольку хостинг будет **GitHub Pages** (только статика), WebApp работает полностью на клиенте. Бот выступает как лёгкий бэкенд для рефералов, синхронизации прогресса и генерации ссылок.

  - WebApp использует `Telegram.WebApp.initDataUnsafe.user.id` для идентификации.
  - Периодическая синхронизация через `sendData()` → бот парсит, обновляет JSON/SQLite, отправляет подтверждение.

# 📦 Инструкция по деплою на GitHub Pages + настройка бота

---

## 1. Создание репозитория и загрузка файлов

```bash
# 1. Создайте новый публичный репозиторий на github.com
#    Название: clicker-webapp (или любое другое)

# 2. Загрузите три файла из папки webapp/:
#    - index.html
#    - style.css
#    - script.js
```

Через веб-интерфейс GitHub:
- Откройте репозиторий → **Add file → Upload files**
- Перетащите `index.html`, `style.css`, `script.js`
- Commit: "Initial clicker webapp"

---

## 2. Включение GitHub Pages

1. Откройте репозиторий на GitHub
2. **Settings → Pages**
3. **Source**: `Deploy from a branch`
4. **Branch**: `main` / `(root)`
5. Нажмите **Save**

Через ~2 минуты страница станет доступна.

---

## 3. Получение ссылки

После активации Pages вверху раздела Settings → Pages появится:

```
Your site is published at:
https://<ваш_username>.github.io/<название_репозитория>/
```

Пример: `https://ivan123.github.io/clicker-webapp/`

---

## 4. Вставка ссылки в бот

Откройте файл `bot/.env`:

```env
BOT_TOKEN=1234567890:AAHxxxxxxxxxxxxxxxxxxxx
BOT_USERNAME=MyClickerBot
WEBAPP_URL=https://ivan123.github.io/clicker-webapp/
```

Перезапустите бота:
```bash
python bot/bot.py
```

---

## 5. Настройка WebApp в @BotFather

```
1. Откройте @BotFather в Telegram
2. Отправьте /mybots → выберите вашего бота
3. Bot Settings → Menu Button → Edit Menu Button URL
4. Вставьте: https://ivan123.github.io/clicker-webapp/

Или создайте отдельное WebApp приложение:
/newapp → следуйте инструкциям → вставьте URL
```

## 6. Ограничения GitHub Pages и обходы

| Ограничение | Решение в проекте |
|-------------|-------------------|
| Только статика (нет серверного кода) | Вся логика в JS + localStorage. Бот — лёгкий бэкенд только для рефералов |
| HTTPS обязателен | GitHub Pages автоматически выдаёт SSL ✅ |
| Кэш браузера задерживает обновления | Версионирование файлов: `style.css?v=2`, `script.js?v=2`. При каждом обновлении меняйте цифру |
| Кэш CDN (до 10 минут) | Добавлены `Cache-Control: no-cache` мета-теги в `<head>` |

При обновлении файлов — увеличьте `?v=` на единицу в `index.html`:
```html
<link rel="stylesheet" href="style.css?v=3">
<script src="script.js?v=3"></script>
```

## 💡 Идеи для масштабирования

1. **Лидерборды** — топ игроков по total_earned, обновляется раз в час (Redis + FastAPI)
2. **Ежедневные бонусы** — streak-система: заходи каждый день за нарастающим бонусом (×1, ×2, ×3...)
3. **Prestige-система** — сброс прогресса за постоянный множитель, открывает новый контент
4. **Защита от автокликеров** — сервер проверяет: кликов/сек не больше физически возможного (≤15/сек), honeypot-паузы
5. **Telegram Stars** — вывод ref_balance через Stars API (`payments.sendStarsForm`), оплата ускорений

## ⚠️ Типичные ошибки

**Кэш браузера на GitHub Pages**
> После обновления файлов изменения не видны. Решение: увеличьте `?v=` в `index.html` и принудительно обновите через Ctrl+Shift+R.

**Утечки памяти в setInterval**
> В `script.js` все интервалы сохранены в переменные (`autoTickInterval`, `autoSaveInterval`, `syncInterval`). При необходимости вызывайте `clearInterval()` при размонтировании.

**Ограничение sendData по размеру**
> `Telegram.WebApp.sendData()` принимает максимум ~4096 байт. Проект отправляет только числа (баланс, клики), не весь JSON сохранения.

**Бот заблокирован при отсутствии @username**
> Все команды кроме `/start` возвращают ошибку с инструкцией. После установки username пользователь снова пишет `/start`.

**Бот не получает web_app_data**
> Убедитесь что WebApp открыт именно через кнопку бота (не браузер). `sendData()` работает только внутри официального Telegram-клиента.
